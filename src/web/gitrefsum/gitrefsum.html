• git init: initializes a directory as a Git repository
  • git clone: copy a git repository so you can add to it
  • git add: adds file contents to the staging area
  • git status: view the status of your files in the working directory and staging area (long output)
  • git status -s: view the status of your files in the working directory and staging area (short output)
  • git diff: show diff of unstaged changes
  • git diff --cached: show diff of staged changes
  • git diff HEAD: show diff of all staged or unstaged changes Add --stat to the above commands to show summary of changes instead of a full diff
  • git commit -m "Message": records a snapshot of the staging area
  • git commit: records a snapshot of the staging area. Git will open an editor for a commit message
  • git commit -a: automatically stage all tracked, modified files before the commit
  • git reset HEAD -- (file): unstage changes that you have staged
  • git branch: list your available branches
  • git branch (branchname): create a new branch
  • git checkout (branchname): switch to a new branch context
  • git checkout -b (branchname): create and immediately switch to a branch
  • git branch -d (branchname): delete a branch
  • git merge (branchname): merge a branch context into your current one
  • git log: show commit history of current branch
  • git log (branchname): show commit history of branch
  • git log --oneline: show compact commit history of a branch
  • git log --oneline --graph: show compact commit history of a branch, with merge history
  • git log --oneline (issue) ^(master): show commits that are in the issue branch, but not in the master branch
  • git log --decorate
  • git tag -a (tag): make an annotated tag
